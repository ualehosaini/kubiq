# Contributing to kubiq

Everyone is welcome to participate in the development of kubiq!

- You are encouraged to read the open issues and implement a solution, then submit a pull request (PR).
- You can also suggest a new issue, implement it, and create a PR for review.
- All contributions will be reviewed by the repository owner before merging.

## Contribution Guidelines (General Best Practices)

- Please discuss any major changes via an issue before submitting a PR.
- Make sure your code is clear, well-documented, and tested.
- Follow the existing code style and project structure.
- Reference the issue number in your PR description if applicable.
- Be respectful and constructive in all communications.

Thank you for helping improve kubiq!
